¿Que es WSL?

Windows Subsystem for Linux (WSL) es una capa de compatibilidad desarrollada por Microsoft para correr ejecutables de Linux (en formato ELF) nativamente en Windows 10 y Windows Server 2019, más información en Wikipedia Esta integración que se hace de Linux dentro de windows nos permite controlar la pc con comandos de linux e instalar programas de linux dentro de windows sin necesidad de virtualizar.

Actualmente puede instalarse desde la página del proyecto WSL, adicionalmente tambien podria usarse BASH dentro de windows permitiendonos usar comandos Linux en windows.

Comandos Basicos

Una buena idea puede ser repasar otros comandos básicos aparte de los aquí listados, por ejemplo en la pagina comandos podemos practicar y entender otros

pwd
Muestra el la ruta del directorio donde nos encontramos posicionados
cd
"Change Directory" Nos permite cambiar de un directorio a otro, si usamos esta instrucción sin ningun parametro adicional, nos enviará al Home directamente, también podemos usar rutas relativas como cd . o cd .. o code ../.. tambien podemos usar rutas absolutas. otra utilidad es que cuando ponemos el principio del nombre del archivo y presionamos tab busca las coincidencias del archivo o carpeta que inicien con el texto ingresado
clear
limpia la pantalla de todos los resultados mostrados en la terminal, opcionalmente puede usarse ctrl+l pero esta instrucción solo recorre el cursor al principio de la pantalla de la terminal, no elimina lo que ha mostradoi anteriormente
mkdir
"Make Directory" permite crear una carpeta en la ubicación en donde nos encontramos
touch
puede crearse un archivo vacío y agregarle una extenión
cat
"Observa" dentro del archivo, mostrándonos su contenido
ls
muestra el contenido de una carpeta

Manipulando Archivos y Directorios

  • Listar: ls y tree
  • Crear: mkdir-Directorios, touch-archivos
  • Copiar: cp
  • Mover y Renombrar: mv
  • Borrar: rm

ls: Listar Directorio

Muestra el contenido de una carpeta, tanto las otras carpetas como los archivos que se encuentran dentro:

Ejemplo:

ziel@ziel-lap:~$ ls
ziel@ziel-lap:~$ ls -a
ziel@ziel-lap:~$ ls -lh
/*muestra los archivos del mas actual al mas antiguo*/
ziel@ziel-lap:~$ ls -lt
/*muestra los archivos del mas antiguo al mas actual*/
ziel@ziel-lap:~$ ls -ltr
/*muestra de forma mas legible el tamaño de los archivos*/
ziel@ziel-lap:~$ ls -lh
/*muestra de forma ordenada los archivos por su tamaño*/
ziel@ziel-lap:~$ ls -lhS
/*muestra los atributos de los archivos y los archivos ocultos*/
ziel@ziel-lap:~$ ls -la

Alguna de sus opciones (argumentos) más útiles son:

  • -a :todos los archivos, incluso los que comienzan con punto (.).
  • -A :Lista todos los ficheros en los directorios, excepto los que comienzan con punto . (.) y los que comienzan con doble punto (…).
  • -F :indica tipo: / directorio, * ejecutable, @ enlace simbólico.
  • -h :indicará el tamaño en KB, MB, etc.
  • -l :listado en formato largo (o detallado).
  • -S :clasifica los contenidos de los directorios por tamaños, con los ficheros más grandes en primer lugar.
  • -r :invierte el orden de la salida.
  • -R :Lista recursivamente los subdirectorios encontrados.
  • -t :ordenar por fecha de última modificación.
  • -u :ordenar por fecha de último acceso.
  • -x :presenta los ficheros por columnas.
  • -i :precede la salida con el número de i-node

tree: Listar Directorio

Nos ayuda a listar un directorio para poder visualizarlo en forma de árbol de forma más estética y ver las relaciones que existen entre ellos, también puede modificarse el nivel de profundidad para mostrar solo lo que nos interesa.

Ejemplo:

ziel@ziel-lap:~$ tree
/*Seleccionando el nivel, en este caso 2*/
ziel@ziel-lap:~$ tree -L 2
/*En caso de no tenerlo instalado*/
ziel@ziel-lap:~$ sudo apt-get install tree
/*Revisar la version*/
ziel@ziel-lap:~$ tree --version

mkdir: Crear un directorio

Para crear un directorio, debemos posicionarnos sobre la carpeta donde serán creados y ejecutar el comando, es recomendable no usar espacios para nombrar a los directorios, pero si es necesarios se debe colocar el nombre entre comillas

Ejemplo:

ziel@ziel-lap:~$ mkdir MiDirectorio
ziel@ziel-lap:~$ mkdir "Mi Directorio"
/*Crea multiples directorios*/
ziel@ziel-lap:~$ mkdir arch1 arch2 arch3 arch4

touch: crea Archivos

touch crea archivos sin ningún contenido, el cual puede agregarse posteriormente, asi mismo podemos definir una extensión al c moomento de crearlo, se puede hacer individual o multiples en una sola línea

Ejemplo:

ziel@ziel-lap:~$ touch archivo.txt
ziel@ziel-lap:~$ touch archivo.txt index.html nose.css

cp: copiar Archivos y Directorios

Este comando nos ayuda a crear la copia de una archivo determinado, si ignoramos la ruta donde se colocará so usará la misma carpeta donde se encuentra el original.

Ejemplo:

/*Crea una copia en el mismo directorio*/
ziel@ziel-lap:~$ cp file fiel_bk
/*Crea una copia en otro directorio*/
ziel@ziel-lap:~$ cp mycoolfile /home/pete/Documents/cooldocs
/*Crea una copia varios archivos a la vez*/
ziel@ziel-lap:~$ cp *.jpg /home/pete/Pictures
/*Crea una copia de la carpeta con todo el contenido*/
ziel@ziel-lap:~$ cp -r Pumpkin/ /home/pete/Documents
/*Crea una copia de la carpeta sin sobreescribir*/
ziel@ziel-lap:~$ cp -i mycoolfile /home/pete/Pictures

mv: mover y Renombrar Archivos y Directorios

Este comando nos ayuda a mover un archivo de un lugar a otro, si no especificamos alguna ruta para hacerlo, se renombrará el archivo en la misma ruta donde se encuentra.

Ejemplo:

/*Renombrar archivo*/
ziel@ziel-lap:~$ mv oldfile newfile
/*Renombrar Carpeta*/
ziel@ziel-lap:~$ mv directory1 directory2
/*Mueve un directorio a la carpeta señalada*/
ziel@ziel-lap:~$ mv file2 /home/pete/Documents
/*Mueve varios archivos a la vez*/
ziel@ziel-lap:~$ mv file_1 file_2 /somedirectory
/*Previene sobreescritura*/
ziel@ziel-lap:~$ mv -i directory1 directory2
/*Crea una copia y coloca ~ al archivo viejo para bakup*/
ziel@ziel-lap:~$ mv -b directory1 directory2

rm: borra Archivos y Directorios

Este comando borra los archivos o carpetas que le indiquemos.

Este comando es peligroso si no se escribe correctamente, ya que puede borrar TODO EL SISTEMA si se usa $rm / dejándolo inutilizable, tampoco se puede dejar un espacio entre el root / y la ruta del directorio o archivo a borrar, ya que lo tomaría como que se borra todo el sistema y después la ruta especificada. Usar $ rm -rf Es muy peligroso por que borra recursivamente, absolutamente todo lo que haya en esa carpeta, sin pasar por la papelera.

Ejemplo:

/*Remueve archivo en la carpeta situada*/
ziel@ziel-lap:~$ rm file1
/*Modo interactivo, pide confirmacion*/
ziel@ziel-lap:~$ rm -i file
/*Borra un directorio con todos sus archivos*/
ziel@ziel-lap:~$ rm -r directory
/*Pregunta uno por uno los elementos a borrar*/
ziel@ziel-lap:~$ rm -ri directory
