¿Que es CSS?

CSS es el acrónimo de Hojas de estilo en cascada (del inglés Cascading Style Sheet ) es el lenguaje de estilos utilizado para describir la presentación de documentos HTML o XML (en-US) (incluyendo varios languages basados en XML como SVG, MathML o XHTML). CSS describe como debe ser renderizado el elemento estructurado en la pantalla, en papel, en el habla o en otros medios.

CSS es uno de los lenguajes base de la Open Web y posee una especificación estandarizada por parte del W3C. Anteriormente, el desarrollo de varias partes de las especificaciones de CSS era realizado de manera sincrónica, lo que permiía el versionado de las recomendaciones. Probablemente habrás escuchado acerca de CSS1, CSS2.1, CSS3. Sin embargo, CSS4 nunca se ha lanzado como una versión oficial.

En resumen, con CSS podemos definir la manera en que lucirá una página web, desde los colores de los elementos, la tipografía e incluso el orden y el espacio de cada uno de los elementos involucrados.

ADVERTENCIA: Para un adecuado entendimiento del manejo de CSS es necesario tener sólidas bases de HTML, de otra forma el aprendizaje será lento, ya que ambas tecnologías se complementan.

CSS en HTML

Una vez que hemos entendido que cada elemento que vamos a modificar debemos definir una regla, ahora debemos entender donde es el lugar adecuado para colocar esa regla y que funcione.

El código CSS se interpreta en el navegador del cliente que abre una página web, para que éste navegador interprete de forma adecuada los estilos que corresponden a un sitio web, debemos especificar como es que cada uno de los elementos que se encuentran dentro de la página deben presentarse y para ello existen 3 formas diferentes de hacerlo.

  • Forma in-line
  • Forma Embebida
  • Forma Enlace
IN-LINE:
Esta forma nos permite colocar el estilo, dentro de los atributos de las etiquetas que se definen en HTML, este atributo solo afecta a la etiqueta en cuestión una a la vez.

Ejemplo:

<p style="font-size: 20px">Mi texto </p>

ADVERTENCIA: Trabajar con estilos In-line es cómodo por que se agregan las reglas dentro de los atributos de la etiqueta, sin embargo, no se aprovecha la cascada de CSS, lo cual es una potente herramienta de diseño, tampoco se pueden repetir los estilos a diferentes páginas del mismo proyecto asi mismo afecta la Especificidad, pues el estilo In-line siempre tiene mayor prioridad sobre las otras formas.

EMBEBIDA:
Esta forma nos permite modificar a los elementos semejantes dentro del mismo archivo HTML, pues al usar un selector, este no se encuentra atado dentro de la misma etiqueta, por lo que se puede aprovechar del efecto en cascada y afectar a otros objetos "Semejantes".

Ejemplo:

<head>
   <style>
      p{font-size:20px;}
   </style>
</head>

ADVERTENCIA: A pesar que la forma embebida nos permite aprovechar de mejor manera la cascada, es una buena práctica utilizar la tercera forma, además que nos permite agregar esas mismas reglas de estilo a otras páginas.

ENLACE:
Esta forma nos permite crear una archivo *.css por separado del archivo *.html, lo que nos permite crear un solo conjunto de reglas que pueden aplicarse a diversas páginas y mantener posteriormente los cambios fácilmente, esta forma de trabajar se considera la mejor práctica para las hojas de estilo.

Ejemplo Enlace:

<head>
     <link rel="stylesheet" href="css/estilo.css" >
</head>

Regla CSS

Debemos entender que una página web es una conjunto de objetos acomodados por etiquetas uno después de otro y cada uno de esos objetos generalmente marcados por estas etiquetas contiene propiedades que pueden alterarse y cambiar el aspecto original de la página. Para que podamos visualizarlos de forma correcta, debemos indicar que propiedades deben modificarse.

La forma en que CSS realiza estos cambios es estableciendo "Reglas Predeterminadas" o simplemente "Reglas" las cuales le indican al nevegador como debe mostrar ese elemento que estamos manipulando, cada regla que establezcamos debe tener la siguiente estructura:

A continuación describimos cada una de sus partes:

SELECTOR:
es la manera en la que nos referimos al elemento al cual aplicaremos la regla que estamos definiendo, para ello, todas las declaraciones que afecten a este elemento debemos escribirlas entre llaves "{...}" para mantenerlas agrupadas, existen muchas maneras de referirnos a esos elementos, si los combinamos adecuadamente, podemos acceder a cualquiera de los elementos establecidos en la página.
DECLARACIÓN:
cada una de las declaraciones modifica una propiedad diferente del elemento seleccionado, cada una de ellas se encuentra compuesta por una propiedad y por el valor que esta asumirá, es importante que al finalizar la declaración, ésta termine con ";".
PROPIEDAD:
La propiedad nos indica que característica del elemento queremos cambiar, su anchura, su altura, su color, etc. de esta manera indicamos al navegador como debe visualizarse. es importante que después de mencionar la propiedad separemos con ":" los valores que le daremos a la misma.

NOTA: Cada propiedad puede llevar consigo más de un valor, generalmente, se identifica un valor de otro por que van separados por espacios, de igual manera existen funciones que nos apoyan para colocar de mejor manera estos valores.

VALOR:
Una vez que hemos señalado la propiedad debemos indicar adecuadamente el nuevo valor que ésta tendrá, este cambio de magnitud depende de la propiedad que estamos modificando, una vez que hayamos establecido los valores adecuados, la declaración debe finalizar con ";".

NOTA: Cada valor es diferente para cada tipo de propiedad, lo más común es que este valor vaya acompañado de un magnitud o unidad de medida que especifica la manera en que la propiedad cambiará.

SELECTORES

Un selector es la manera con la cual nosotros nos referimos a un elemento de la página web a la cual nosotros aplicaremos una serie de declaraciones que modificarán su aspecto final, es importante conocer la forma de dirigirnos a ese elemento, por que si lo hacemos de forma incorrecta, los estilos no se aplicaran correctamente, para ello, existe un amplia diversidad de herramientas que podemos utilizar y combinar entre ellas para seleccionar correctamente el elemento.

A continuación enumeramos los tipos de selectores que disponemos.

  • Selector de Tipo
  • Selector de Clase
  • Selector de ID
  • Selector de Atributo
  • Selector de Pseudoclases
  • Selector de Pseudoelementos
  • Selector Universal

De la misma forma disponemos de operadores de combinación, los cuales nos permiten ampliar de forma significativa nuestras herramientas de selección, entre ellas encontramos:

  • Combinador coma ","
  • Combinador Espacio" "
  • Combinador suma"+"
  • Combinador virgulilla"~"

Entendamos como funcionan cada uno de ellos:

SELECTOR DE TIPO
Un selector de tipo también recibe el nombre de selector de nombre, de etiqueta, selector de elemento o selector general porque selecciona un elemento/etiqueta HTML del documento de forma directa.

En archvo HTML:

<p>Mi texto </p>

En archvo CSS:

p{font-size: 20px;}

ADVERTENCIA: cuando usamos esta forma de seleccionar los elementos, la regla que definamos se aplica para todas las etiquetas del mismo tipo. En el caso del ejemplo todas las etiquetas "p" seran afectadas por este estilo.

SELECTOR DE CLASE
Este selector de clase nos proporciona un metodo para seleccionar a solo "algunos" elementos a quienes deseamos aplicar la regla, es decir, solo los elementos que pertenecen a la clase que definimos pueden ser seleccionados, en este caso, no importa si los elementos son de etiquetas diferentes o etiquetas iguales.

La forma correcta de implementarlo es asignadole una clase en el atributo "class" en las etiquetas del archivo *.html de esa manera "marcamos" las etiquetas a las cuales se le aplicaran la regla que definiremos en el archivo *.css, cuando definamos la regla el nombre de la clase debe estar precedida de "." para indicar que es el nombre de la clase.

En archvo HTML:

<p class="amarillo"> Mi texto </p>

En archvo CSS:

.amarillo{color: yellow;}
De ser necesario, podemos modificar el ejemplo para asignar la clase a diferentes etiquetas y observar como se aplica la regla a las etiquetas que seleccionamos, miestras que las etiquetas que no contienen a la clase, quedan excluidas de la regla, todo esto se aplica sin modificar el archivo *.css.

En archvo HTML:

<h1 class="amarillo"> Titulo Amarillo </h1>
<p> Este parrafo va en color predeterminado</p>
<p> Este parrafo tampoco se aplica la regla/p>
<p class="amarillo"> Este parrafo SI es amarillo</p>
De la misma forma podemos agregar más de una clase a las etiquetas, lo que nos permite agregar más características a un elemento. eso lo podemos ver en este ejemplo:

En archvo HTML:

<!-- este titulo llevaria caja alrededor y letra amarilla-->
<h1 class="amarillo caja"> Titulo Amarillo </h1>
<p> Este parrafo va en color predeterminado</p>
<p class="caja"> Este parrafo tampoco se aplica la regla pero tiene un borde negro alrededor</p>
<p class="amarillo"> Este parrafo SI es amarillo</p>

En archvo CSS:

.amarillo{color: yellow;}
.caja{border: thin solid black;}
SELECTOR DE ID
El Selector Id o de referencia absoluta nos ayuda a seleccionar directamente un único elemento que en su etiqueta html contenga el atributo "id". para referirnos desde el archivo *.css es necesario que se anteponga el caracter "#"

En archvo HTML:

<p id="amarillo"> Mi texto </p>

En archvo CSS:

#amarillo{color: yellow;}
SELECTOR DE ATRIBUTO
El Selector de Atributo nos permite elegir un elemento de la página web al cual le aplicaremos una regla, siempre y cuando se haya especificado por lo menos algún atributo, podemos seleccionar el elemento por el tipo de atributo, por el valor preciso del atributo o por alguna frase que pertenezca al valor del atributo, la nomenclatura es la siguiente:

Presencia y Valor
Selector Ejemplo
[attr] a[title]{/*...*/}
DESCRIPCION: Selecciona a los elementos o etiquetas del archivo *.html que contengan un atributo concreto, sin importar su valor. reemplazamos attr por el nombre del atributo que buscamos entre los corchetes.
[attr=value] a[href="https://example.com"] {/*...*/}
DESCRIPCION: Selecciona elementos que tiene un atributo de nombre attr, y su valor exacto es value (la cadena de caracteres que se indica entre corchetes).
[attr~=value] p[class~="special"]{/*...*/}
DESCRIPCION: Se lecciona a las etiquetas del archivo *.html con el atributo de nombre attr, cuyo valor es exactamente value, o los elementos con un mismo atributo attr que contiene uno o más valores de los cuales, al menos uno, coincide con value. Hay que tomar en cuenta que los diferentes valores generalmente se separan por espacios.
[attr|=value] div[lang|="zh"]{/*...*/}
DESCRIPCIÓN: Relaciona a los elementos con un nombre de atributo, attr, cuyo valor puede ser exactamente value o puede comenzar con value seguido inmediatamente por un guion.


Coincidentes con subcadenas
Selector Ejemplo
[attr^=value] li[class^="box-"]{/*...*/}
DESCRIPCIÓN: Relaciona a los elementos con un nombre de atributo, attr, cuyo valor comienza exactamente con value.
[attr*=value] li[class*="box"]{/*...*/}
DESCRIPCIÓN: Relaciona a los elementos con un nombre de atributo, attr, cuyo valor contiene al menos una ocurrencia de la subcadena value.
[attr$=value] li[class$="-box"]{/*...*/}
DESCRIPCIÓN: Relaciona a los elementos con un nombre de atributo, attr, cuyo valor termina exactamente con la subcadenavalue.


Case sensitive
Selector Ejemplo
i li[class^="a" i]{/*...*/}
DESCRIPCIÓN: Selecciona los elementos que cumplan con la descripción del selector pero adicionalmente su valor sea en letras minusculas
s ol[type="A" s]{/*...*/}
DESCRIPCIÓN: Selecciona los elementos que cumplan con la descripción del selector pero adicionalmente su valor sea en letras minusculas, este atributo es experimental.
SELECTOR DE PSEUDOCLASE
Las Pseudoclases son palabras claves que se añaden a los selectores, los cuales representan un estado en particular del elemento seleccionado. Todas las pseudoclases se comportan del mismo modo. Seleccionan un fragmento del documento que está en un estado determinado y se comportan como si se hubiera añadido una clase a su html.

Ejemplo:

article p:first-child{/*...*/}
a:hover{/*...*/}
a:visited{/*...*/}
Algunas pseudoclases utiles son las siguientes:
  • :nth-child(): Permite seleccionar un hijo de un elemento que tiene varios hijos, cada hijo se enumera según el orden de aparición, lo que nos permite colocar ese número ordinal entre paréntesis para ubicarlo, también es posible usar las palabras "odd" y "even" para indicar elementos impares o pares, según sea necesario
  • :first-child: Selecciona al primer hijo de un elemento
  • :last-child: Selecciona al último hijo
  • :only-child: Selenciona al que tiene solo un hijo
  • :root: Representa a la etiqueta Html, pero tiene mayor especificidad, por lo cual predomina. generalmente se usa para declarar variables.
  • :hover: se activa al pasar el mouse por encima
  • :active: se usa este estado en el momento que se mantiene presionado el mouse, para que funcione debe ir en orden las reglas con las pseudoclases, :link{}, :visited{}, :hover{}, :active
  • :target
  • :not
  • :empty
SELECTOR PSEUDOELEMENTO
Los Pseudoelementos se comportan como si agregaramos un elemento html nuevo al marcado, para diferenciarlos de las pseudoclases agregamos "::" al selector y el nombre del pseudoelemento en estaLista de Pseudoelementos y Pseudoclases podemos ver una lista de ellos.

Ejemplo:

article::before{/*...*/}
p::after{/*...*/}
p::first-letter {/*...*/}
a::first-line{/*...*/}

ADVERTENCIA: para que podamos usar un ::before o un ::after debemos agregar un valor a "content" aunque sea un valor como content:""; para que pueda verse el elemento y las modificaciones que le hagamos a estos pseudoelementos. Estos pseudoelementos no funcionan en los elementos in-line

SELECTOR UNIVERSAL
El selector universal se indica con un asterisco (*) y selecciona todos los elementos del documento (o del elemento padre si está encadenado con otro elemento y un operador de combinación descendente, por ejemplo). *{margin: 0;} Es posible observar este tipo de comportamiento en las «hojas de estilo de puesta a cero» (o «hojas de estilo reset»), que anulan el formato del navegador.

Uno de los usos del selector universal es facilitar la legibilidad de los selectores y clarificar sus funciones. Por ejemplo, si quiero seleccionar el primer elemento hijo de cualquier elemento <article> y poner ese elemento, cualquiera que sea, en negrita, puedo utilizar el selector :first-child, que veremos con mayor detalle más adelante en el artículo de pseudoclases y pseudoelementos, como selector descendente junto con el selector de elemento <article>:

article :first-child {/*...*/}

Sin embargo, esto podría confundirse con article:first-child, que selecciona cualquier elemento <article> que sea el primer elemento hijo de otro elemento.

Para evitar esta confusión podemos añadir al selector :first-child el selector universal. De este modo la función del selector resulta obvia: seleccionará cualquier elemento que entre en la jerarquía de primer hijo de un elemento <article>:

article *:first-child {/*...*/}

Herencia, Cascada y Especificidad

Estos 3 conceptos se encuentran estrechamente relacionados y son los responsables que se apliquen o no las reglas que se definen, estas características tambien son útiles para asignarle la misma regla a diferentes elementos con características similares sin necesidad de repetir código

Herencia

La herencia es el mecanismo mediante el cual un objeto padre (o de jerarquía superior) "hereda" características a sus hijos directos y ellos a su vez heredarían éstas características o lo siguientes elementos descendientes. Existirán características que serán heredables y algunas otras no podrán heredarse.

CSS proporciona 4 valores (1 elemento experimental) de propiedad universales especiales para el control de la herencia. Todas las propiedades CSS aceptan estos valores.

inherit:
Este valor hace que la propiedad a la que se aplica, sea exactamente igual al valor de su padre en la misma propiedad, en otras palabras "activa la herencia"
initial:
Cada navegador tiene por defecto una hoja de estilo para los elementos, la cual suele ser diferente para uno de ellos, esta hoja posiciona esos valores básicos, al usar el valor "initial" la propiedad regresa a los valores por defecto que posee el navegador
unset:
Reestablece el valor a su valor natural, puede ser inherit o initial, dependiendo de la propiedad a la que se aplique.
revert:
es un valor experimental, el cual tendría como propósito revertir la cascada y regresar su valor al que heredaría de su ancestro más cercano
all
Esta es una Propiedad abreviada que tiene por objetivo controlar la herencia en las reglas CSS, esta afecta a casi todas las propiedades a la vez y puede tomar cualquiera de los 4 valores anteriores por ejemplo: a {all:revert;}

Cascada

Esta propiedad permite que un regla pueda aplicarse a diferentes elementos, por ejemplo, si definimos una regla para los parrafos "p" todas las etiquetas "p" aplicaran la misma regla, sin embargo, si mas adelante se enuncia una regla diferente para el selector "p", se aplicará la ultima regla que aparezca:

Ejemplo:

/*...*/
p{color:blue;}
p{color:red;}
p{color:orange;}
/*...*/

En este ejemplo podemos ver visualmente como es que se han aplicado diferentes reglas a las etiquetas "p", sin embargo la regla que prevalecerá será la que tiene color naranja, ya que la cascada respeta la última regla válida.

Especificidad

La especificidad, se aplica para aquellos elementos se encuentran referidos en más de una regla y tienen diferente selector, para ello, se analiza la manera en que el selector se ha construido y se refiere a ese elemento, dándole valores a las diferentes formas de construir el selector, provocando que aquel que sea más específico para referirse al elemento sea quien prepondere sobre los demás.

Para explicar mejor esta idea, se le da un valor numérico a los selectores, se distingue entre etiquetas, clases, id, inline y sobre el important ese valor se suma y se aplica la regla que tenga un selector con mayor número, es decir, mayor especificidad.

Los valores aproximados que se asignan son los siguientes.

Tag 1 El selector de tipo de etiqueta "<>", o pseudoelemento "::" agrega una unidad.
Clase 10 El selector de clase ".", de atributo"[]" o pseudoclase ":"
Id 100 Para cada selector de Id "#"
In-Line 1000 Estos estilos se especifican directamente sobre el atributo "style" de la estiqueta correspondiente en el documento html
!important 1,000,000 la declaración que contenga como valor !important prevalece sobre cualquier otra regla

ADVERTENCIA: Usar "!important" es una mala práctica, conocer este valor para los atributos es útil para casos extremos, pero no para casos comunes que pueden resolverse con clases o combinadores.

Para entender mejor esta idea, es útil que entendamos como es que el navegador asigna estilos a los diferentes elementos.

  1. Herencia
  2. Orden del código
  3. Especificidad
  4. Importancia
Herencia:
El navegador primero se fija en la hoja de estilos que tiene predeterminada y la forma en que estos estilos se heredan a sus hijos, estos se colocan de forma inicial
Órden del código:
Después observa los estilos que se encuentran definidas en la hoja de estilos del usuario, si hay conflicto, las hojas definidas por el usuario tienen preponderancia, si hay diferentes reglas que se aplican al mismo elemento se observa el orden en que aparecen y las últimas definidas son las que tienen mayor preponderancia
Especificidad:
Si existen coincidencias de reglas que se refieren a un mismo elemento, el navegador observa que regla se ha definido de forma "más específica" es decir, cual regla provoca que se seleccione a un elemento de manera inequivoca o de forma menos general.
Importancia
Propiedades que contengan el valor !important

Modelo de Caja

Flujo Normal

Lo que se conoce como flujo normal es la manera en que los diferentes elementos se acomodan en el navegador de forma predeterminada, es decir, sin que exista una hoja de estilos que le ordene como deben verse y acomodarse esos elementos.

Esta manera de acomodar a los elementos, es útil por que nos da un inicio para no trabajar desde cero el acomodo y arreglo de los diferentes elementos, esta primera forma respeta el orden en que se describieron las etiquetas en la página html y se acomodan una atras de otra respetando la naturaleza de cada etiqueta, esta naturaleza puede ser en Bloque o en Línea

Bloque:
Las etiquetas en bloque tienen la característica que se apilan de forma vertical una sobre otra, éstas etiquetas pueden contener otras etiquetas de bloque u otras etiquetas en línea, en este enlace aparecerá una lista de etiquetas en bloque. algunas de sus características son:
  • La caja fuerza un salto de línea al llegar al final de la línea.
  • La caja se extenderá en la dirección de la línea para llenar todo el espacio disponible que haya en su contenedor. En la mayoría de los casos, esto significa que la caja será tan ancha como su contenedor, y llenará el 100% del espacio disponible.
  • Se respetan las propiedades width y height.
  • El relleno, el margen y el borde mantienen a los otros elementos alejados de la caja.
In-line:
Las etiquetas con naturaleza In-line se acomodan en el eje horizontal una a lado de la otra, una forma de visualizarlo sería verlos como si fueran "palabras" dentro de un párrafo, estas palabras solo ocupan el espacio absolutamente necesario y tienen una separación mínima entre otras palabras, esta es una lista de elementos en línea
  • La caja no fuerza ningún salto de línea al llegar al final de la línea.
  • Las propiedades width y height no se aplican.
  • Se aplican relleno, margen y bordes verticales, pero no mantienen alejadas otras cajas en línea.
  • Se aplican relleno, margen y bordes horizontales, y mantienen alejadas otras cajas en línea.

Existen dos tipos de visualizaciones, la Externa y la interna, la externa tiene que ver directamente con su naturaleza, si es en bloque o in-line, sin embargo aunque la visualización interna sigue este patron, esta visualización interna puede cambiarse por ejemplo al usar display:flex o display:grid

Display

La manera en que podemos cambiar la naturaleza de una caja es mediante la propiedad display:/*valor*/; formalmente esta propiedad tiene la capacidad cambiar tanto la visualización interna y externa de los elementos y existe una extensa aplicacion que puede hacerse gracias a esta propiedad y puede verse en esta Lista de valores display, sin embargo, por ahora mostraremos solo 3 valores:

block

Modelo de cajas

Una vez que hemos entendido el flujo natural de los elementos, tenemos que observar cuales son las propiedades que afectan el tamaño de ellos, para después "acomodarlas" en un proceso que se conoce como maquetado, estas propiedades se superponen una encima de la otra "Engordando" a la caja, lo que provoca que se afecte el tamaño final de la caja, esta situación puede provocarnos comportamientos inesperados al momento de acomodar los elementos. Los elementos que se ven afectados por todas las propiedades que a continuación se enumeran son los de bloque, que los in-line, solo se ven afectados por algunos de ellos:

  • El contenido de la caja (o content box): El área donde se muestra el contenido, cuyo tamaño puede cambiarse utilizando propiedades como width y height.
  • El relleno de la caja (o padding box): El relleno es espacio en blanco alrededor del contenido; es posible controlar su tamaño usando la propiedad padding y otras propiedades relacionadas.
  • El borde de la caja (o border box): El borde de la caja envuelve el contenido y el de relleno. Es posible controlar su tamaño y estilo utilizando la propiedad border y otras propiedades relacionadas.
  • El margen de la caja (o margin box): El margen es la capa más externa. Envuelve el contenido, el relleno y el borde como espacio en blanco entre la caja y otros elementos. Es posible controlar su tamaño usando la propiedad margin y otras propiedades relacionadas.

Modelo de Caja Estandar

cuando se definen las propiedades de width y height, estas se aplican directamente al content-box, y todas las demás propiedades se suman de forma externa para obtener el tamaño final. Por ejemplo:

.box {
width: 350px;
height: 150px;
margin: 10px;
padding: 25px;
border: 5px solid black;
}

como se puede apreciar, el espacio que ocupa nuestra caja usando el modelo de cajas estándar será en realidad de 410 px (350 + 25 + 25 + 5 + 5); y su altura, de 210 px (150 + 25 + 25 + 5 + 5), porque el área de relleno y el borde se añaden al ancho que se utiliza para el contenido de la caja

Nota: El margen no se cuenta para el tamaño final de la caja, el tamaño de la caja se calcula hasta el borde

Modelo de Caja Alternativo

El este modelo de caja el tamaño dado por los valores width y height se determina por la caja del borde, esto permite que los valores de padding y border queden "dentro" del tamaño de caja definido, por lo que sabremos a ciencia cierta cuanto mide la caja y no debemos de añadir mas pixeles.

Para lograrlo, debemos agregar la propiedad box-sizing con el valor border-box ya que los navegadores de forma predeterminada usan el modelo de cajas estandar, y debemos "decirle" que active esta característica para ayudarnos a tener un mejor control en el tamaño.

.box {
box-sizing: border-box;
/*...*/
}

Si quieres que todos tus elementos usen el modelo de cajas alternativo (opción común entre los desarrolladores) debes establecer la propiedad box-sizing en el elemento <html>. Luego debes configurar todos los demás elementos para que hereden ese valor, como se ve en el fragmento de código siguiente. Si deseas comprender qué hay detrás, consulta el artículo de CSS-Tricks sobre el tamaño de las cajas.

html{
box-sizing: border-box;
}
*, *::before, *::after{
box-sizing: inherit;
}

    Es una función que nos permite colocar la dirección en la que se encuentra la imagen que deseamos colocar, cabe mencionar que la dirección puede ser relativa, es decir en la misma carpeta donde almacenamos la página web o una dirección web específica
    none
    Utilizado para especificar que un elemento no debe tener ninguna imagen de fondo.

    Ejemplo:

    background-image: url("https://mdn.mozillademos.org/files/6457/mdn_logo_only_color.png");
    background-image: url("https://mdn.mozillademos.org/files/11991/startransparent.gif"), url("https://mdn.mozillademos.org/files/7693/catfront.png");

    background-repeat

    El comportamiento de "mosaico" que se observa en los elementos a los que se agrega una imagen puede controlarse con background-repeat, este control se ejerce sobre el eje horizontal o el eje vertical o ambos a la vez, de acuerdo a los siguientes valores que acepta.

    Sintaxis:

    background-repeat: repeat | no-repeat | repeat-x | repeat-y | space | round | < horizontal > < vertical > ;
    repeat
    Es la opcion por defecto, puede usarse en forma horizontal (repeat-x) o vertical (repeat-y) La imagen se repite hasta cobrir todo el fondo existente. La última imagen debe ser recortada si no encaja.
    space
    La imagen se repite lo máximo posible sin recortarse. La primera y última imagen son fijadas a cada lado del elemento, y el espacio blanco es distribuido igualmente entre las imágenes. La propiedad background-position es ignorada a menos que una sola imagen pueda ser desplegada sin recortar. El único caso donde se recorta usando space es cuando no hay suficiente sitio para desplegar una imagen.
    round
    Como el espacio permitido aumenta, las imágenes repetidas se estrechan (sin dejar huecos) hasta que haya espacio suficiente (espacio restante >= la mitad del ancho de la imagen) para que otra sea añadida. Cuando la próxima imagen es añadida, todas las demás son comprimidas al espacio disponible. Ejemplo: Una imagen con un ancho inicial de 260px, se repite 3 veces, debería estirarse hasta que el tamaño sea de 300px, luego otra imagen debe ser añadida. Luego deberían comprimirse hasta los 225px.
    no-repeat
    La imagen no se repite (y por lo tanto el area coloreada de la imagen de fondo no debe ser rellenada completamente). La posición del fondo no repetido es definida por la propiedad de CSS background-position.

    Ejemplo:

    /* One-value syntax */
    background-repeat: repeat-x;
    background-repeat: repeat-y;
    background-repeat: repeat;
    background-repeat: space;
    background-repeat: round;
    background-repeat: no-repeat;

    /* Two-value syntax: horizontal | vertical */
    background-repeat: repeat space;
    background-repeat: repeat repeat;
    background-repeat: round space;
    background-repeat: no-repeat round;

    /* Global values */
    background-repeat: inherit;
    background-repeat: initial;
    background-repeat: unset;

    background-size

    La propiedad background-size nos permite controlar el tamaño que tendrá la imagen dentro del elemento contenedor, de esta manera si la imagen es mayor al contenedor, esta podrá reducirse o en caso contrario ampliarse hasta que se adapte a nuestras necesidades.

    Sintaxis:

    background-size: width=height | width height | contain | cover | auto ;
    auto
    Es el valor por defecto y mantiene las proporciones y medidas originales de la imagen.
    width=height
    Es un valor numérico que se aplica a las dos dimensiones de la imagen por igual, puede usarse un porcentaje o una medida determinada en pixeles o relativa
    width height
    Podemos especificar por separado los dos valores tanto horizontal como vertical para tener un mejor control de la imagen
    contain
    La imagen mantiene su proporcion, y se adapta el tamaño hasta que esta quede "contenida" dentro del elelemento, el navegador adecuará el tamaño de la imagen para que quepa dentro de la caja. En este caso, pueden quedar huecos a ambos lados o en la parte superior e inferior de la imagen si la relación de aspecto de la imagen es diferente de la de la caja.
    cover
    Con esta opcion la imagen mantiene su proporción, pero es estirada hasta que cubra completamente al contenedor, de esta forma, parte de la imagen puede quedar recortada por su elemento contenedor.

    Ejemplo:

    /* NO RECOMENDADO */
    background-size: 25px;
    background-size: 50%;
    background-size: auto 50px;
    background-size: auto 50%;

    /* OKAY */
    background-size: 25px 50px;
    background-size: 50% 50%;

    background-position

    La posicion de la imagen se determina por la propiedad background-position la imagen de forma determinada se posiciona en la coordenada (0,0) que se encuentra en la esquina superior izquierda, tomando el sentido positivo de izquierda a izquierda en horizontal y arriba a abajo en vertical.

    Sintaxis:

    background-position: [ <percentage> | <length> | left | center | right ]
    [ <percentage> | <length> | top | center | bottom ] ? ;
    background-position: [ top | center | bottom ];
    background-position: inherit;
    <percentage>
    Con el par de valores '0% 0%', la esquina izquierda de la imagen es alineada con la esquina izquierda del borde del espaciado de la caja. Un valor '100% 100%' pone la esquina inferior de la imagen en la esquina inferior del área de espaciado. Con un valor '14% 84%', el punto a 14% a la derecha y 84% abajo de la imagen es colocado en el punto al 14% derecha y 84% abajo del área de espaciado.
    <length>
    con un valor '2cm 1cm', la esquina superior izquierda de la imagen es colocada a 2 cm a la derecha y a 1cm debajo de la esquina superior izquierda del área de espaciado.

    Ejemplo:

    background-position: top center;
    background-position: 20px 10%;
    background-position: top 20px;
    background-position: top 20px right 10px;

    background-clip

    La propiedad background-clip provoca que la imagen que se ha ubicado en el fondo sea cortada por una de las cajas del modelo de caja.

    Sintaxis:

    background-clip: border-box | padding-box | content-box | inherit ;
    border-box
    El fondo se extiende hasta el borde exterior de el contenedor (pero por debajo de la frontera, en orden z).
    padding-box
    El fondo no se muestra a través del borde (el fondo se extiende hasta el borde exterior del padding).
    content-box
    El fondo se dibuja desde donde inicia el contenido.

    The yellow background extends behind the border.

    The yellow background extends to the inside edge of the border.

    The yellow background extends only to the edge of the content box.

    Ejemplo:

    background-clip: border-box
    background-clip: padding-box
    background-clip: content-box

    background-clip: inherit

    background-origin

    Con la propiedad background-origin nosotros definimos a partir de que caja del modelo de caja se comienza a dibujar la imagen.

    Sintaxis:

    background-origin: padding-box | border-box | content-box ;
    padding-box
    El fondo se extiende al borde exterior (por debajo del borde en el orden z), la posición es relativa al borde de la caja.
    border-box
    El fondo se extiende al borde exterior del padding, la posición es relativa al padding de la caja.
    content-box
    El fondo se extiende dentro del (recortado al) contenido de la caja, la posición es relativa al contenido de la caja

    Ejemplo:

    background-origin:border-box;
    /*Multiples Background*/
    background-origin: padding-box, content-box;

    ADVERTENCIA: La propiedad background-origin es ignorada si la propiedad background-attachment tiene un valor de fixed.

    Si el valor de esta propiedad no es establecido en la propiedad abreviada (shorthand) background que es aplicada a el elemento despues de la propiedad background-origin, el valor de esta propiedad sera restablecido a su valor inicial por la propiedad abreviada background.

    background-attachment

    Otra opcion del fondo de los elementos es controlar la manera en que la imagen se desplazará al momento de desplazarse a través de la pagina, es decir de hacer scroll, esto se controla con background-attachment

    Sintaxis:

    background-attachment: scroll | fixed | local | inherit
    scroll
    Es el valor por defecto y hace que el fondo del elemento se desplace cuando te desplazas por el contenido de la página. Si te desplazas por el contenido del elemento, el fondo no se mueve. El efecto resultante es que el fondo está anclado a la página en una posición fija, por lo que se desplaza a medida que la página se desplaza.
    fixed
    Hace que el fondo de un elemento quede fijo con respecto a la ventana gráfica, de modo que no se desplace cuando te desplazas por la página o el contenido del elemento. Siempre permanece en la misma posición de la pantalla.
    local

    Ejemplo:

    background-attachment: fixed;
    /*Multiple*/
    background-attachment: fixed, scroll;

    Multiples Fondos

    Es posible usar mas de un fondo o multiples imagenes dentro de un mismo objeto, para ello debemos de usar las propiedades background-* y separarlas por comas, Si haces esto, las imágenes de fondo pueden quedar superpuestas entre sí. Los fondos se superponen con la última imagen de fondo que hay en la parte inferior de la lista, y cada imagen anterior se apila encima de la que sigue en el código.

    NOTA: Los degradados se pueden mezclar con imágenes de fondo normales, las imagenes se colocan en orden, y si estas tienen transparencias pueden crearse efectos interesantes.

    Ejemplo:

    background-image: url(image1.png), url(image2.png), url(image3.png), url(image1.png);
    background-repeat: no-repeat, repeat-x, repeat;
    background-position: 10px 20px, top right;

    Cada valor de las diversas propiedades coincide con los valores que están en la misma posición en las otras propiedades. Arriba, por ejemplo, el valor para la propiedad background-repeat de image1 será no-repeat. Sin embargo, ¿qué sucede cuando diferentes propiedades tienen una cantidad diferente de valores? La respuesta es que los valores que ocupan las posiciones más pequeñas se alternan cíclicamente: en el ejemplo anterior hay cuatro imágenes de fondo pero solo dos valores background-position. Los primeros dos valores de posición se aplicarán a las dos primeras imágenes, luego los valores volverán a asignarse cíclicamente: a image3 se le dará el primer valor de posición, y a image4 se le dará el segundo valor de posición

    Overflow

    El desbordamiento de los elementos que se encuentran en un contenedor, puede darse por que el texto ocupa mayor espacio que el tamaño definido del contenedor, es decir cuando el contenedor no fluye con el redimensionamiento de la página y por el contrario tiene una medida definida, en el caso del texto podemos manejarlo con overflow, Los elementos de elementos de reemplazo pueden manejarse de otra forma, como se muestra en este artículo podemos usar la propiedad max-width u object-fit

    A pesar de que se pueda definir el tamaño de una caja desde CSS, siempre existe la posiblidad que el contenido no se ajuste a este tamaño ya sea por que se vea mas pequeño o por que el contenido supere el tamaño del contenedor, es en este momento en que llamamos un desborde de elementos.

    De manera predeterminada, la forma en que CSS gestiona el desbordamiento es que "deja salir" el contenido, es decir, se muestra fuera de la caja, esto lo hace para evitar problemas de visualización de contenido y se oculte información al usuario que pueda ser importante. La forma adecuada con lidiar con el desbordamiento es usando la propiedad overflow.

    /*Valor predeterminado*/
    overflow: visible
    /*Valores*/
    overflow: hidden
    overflow: scroll
    overflow: auto
    overflow: inherit
    visible
    Valor por defecto. El contenido no es recortado, podría ser dibujado fuera de la caja contenedora.
    hidden
    El contenido es recortado y no se muestran barras de posición.
    scroll
    El contenido es recortado y el navegador web usa las barras de desplazamiento, se haya recortado contenido o no. Esto previene cualquier problema con las barras de desplazamiento apareciendo o desapareciendo en un entorno dinámico. Puede que las impresoras impriman contenido excedente.
    auto
    Depende del agente de usuario. Navegadores de escritorio como Firefox proveen barras de desplazamiento si hay contenido excedente.

    En el caso que se quiera controlar solo el scroll vertical u horizontal, podemos usar la propiedad overflow-y y overflow-x

    Nota: En un contenedor fluido relleno de texto, el tamaño mínimo depende de la palabra más larga, en el caso de lidiar con palabras muy largas que provoquen el desbordamiento se puede usar la propiedad word-break o overflow-wrap para solucionar el problema.

    Advertencia: En CSS hay un concepto conocido como block formatting context o BFC (contexto de formato de bloque). Cuando usas un valor de overflow, como scroll o auto, creas un BFC. El resultado es que el contenido de la caja al que acabas de cambiar el valor overflow se convierte en un minidiseño propio. Las cosas que están fuera del contenedor no pueden meterse en él, y nada puede sobresalir de esa caja hacia el diseño circundante. Esto es para permitir un comportamiento con desplazamiento, porque para poder crear una experiencia de desplazamiento consistente todo el contenido de tu caja ha de estar contenido en algo, y no puede superponerse con otros elementos de la página.

    Flujo Normal

    La manera "Normal" en la que los elementos se acomodanen la página web sin ninguna intervención o modificación, obedece a la naturaleza de cada uno de los elementos que tienen por defecto, es en este sentido que el modelo de caja toma relevancia, ya que estos elementos se acomodan uno encima de otro o a lado de sus compañeros, dependiendo si su naturaleza es en línea o en bloque.

    Sin embargo este comportamiento puede ser cambiado a conveniencia, cambiando el flujo normal y forzar el comportamiento de los diferentes elementos para que estos se muestren en la manera que nosotros necesitamos.

    Si por algún motivo, nuestra página no puede cargar el css, nuestra página se verá de forma básica como la hayamos estructurado en HTML, de esta manera, una buena estructura desde el HTML hará que esta página se muestre de forma legible para el usuario aunque los estilos no estén presentes.

    Los bloques se comportan de acuerdo al orden de lectura con el que se encuentre configurado el navegador, en occidente el orden de lectura es de izquierda a derecha y de arriba a abajo, en otros países el comportamiento de los bloques puede ser diferente.

    Display y Visibility

    una forma de modificar el comportamiento de las cajas dentro de la maquetación es usar la propiedad visibility, esta propiedad nos permite "hacer transparente" al objeto a quien lo estamos aplicando, sin embargo el lugar que deja vacío se mantiene y provoca que los demás objetos "respeten el espacio que le corresponde" dejando un hueco.

    Sintaxis Visibility:

    visibility: visible | hidden | collapse | inherit;
    visible
    El elemento se muestra normalmente.
    hidden
    El elemento está escondido, pero los demás elementos se colocan como si ése elemento estuviera presente. Esto funciona como si el elemento fuera absolutamente transparente. Los elementos, con la propiedad visibility: visible, descendientes de ése elemento serán visibles.
    collapse
    Para tablas, filas, columnas y grupos de tablas. Las filas o columnas se esconden y el espacio que normalmente ocupan, es ocupado (como si aplicáramos el código display: none a las filas/columnas de la tabla). Sin embargo, el tamaño de las columnas o filas restantes, es calculado como si las filas/columnas fusionadas estuvieran presentes. Esta propiedad se diseñó así, al fin de permitir la supresión rápida de filas/columnas de una tabla sin necesidad de calcular de nuevo los altos y los anchos de cada porción de tabla. (Para otros elementos, collapse funciona como hidden).

    Ejemplo:

    p{ visibility: hidden; } /* Los párrafos no serán visibles*/
    p.showme { visibility: visible; } /* excepto los que tengan class = showme (clase= muéstrame)*/
    tr.col { visibility: collapse; } /* las filas de la tabla con la clasecol se fusionarán*/

    El soporte para visibility: collapse no se encuentra o está implementado parcialmente en algunos navegadores modernos. En muchos casos será tratado como visibility: hidden en elementos que no son filas / columnas de una tabla.

    visibility:collapse puede cambiar el aspecto de una tabla si hay anidadas tablas dentro de celdas fusionadas, a no ser que se especifique explícitamente visibility: visible en la tabla anidada.

    En cambio, si deseamos que el objeto se oculte, pero que las demás cajas, tampoco respeten el lugar que a este le pertenecía, lo más adecuado es usar display, usando la propiedad display: none;, aunque la funcionalidad de display no está para nada limitada a solo esta característica, a través de ella obtenemos mas herramientas de maquetación aún más poderosas, algunas de ellas pueden ser:

    Float

    Este estilo de maquetación utiliza la propiedad float y se usa por compatibilidad de navegadores viejos, su función original era flotar los elementos o imagenes dentro de los bloques de texto, similar a como se usan las letras capitales en los periódicos, sin embargo por mucho tiempo se convirtió en una herramienta de maquetación hasta la llegada de Flexbox y Grid. podemos ver la sintaxis

    none
    El elemento no deberá flotar, cabe mencionar que esta propiedad no elimina el hecho que el elemento que estaba flotando deje de hacerlo

    Para limpiar el efecto, se puede usar la propiedad clear en el elemento o parrafo siguiente,la forma normal de hacerlo es que el cuadro contenedor de los párrafos y los elementos flotados utilicen el "hack clearfix"

    Clearfix


    .wrapper::after {
    content: "";
    clear: both;
    display: block;
    }

    John Mircha recomienda lo siguiente, tomando la idea de que multiples elementos, anteriores y posteriores usaran la flotación para maquetar, por lo tanto, se necesita que los pseudoelementos, after y before sean los que limpien esta propiedad.

    Clearfix


    .clear, .clear::after, .clear::before{clear: both;}
    .clear::after, .clear::before{
    content: "";
    display: block;
    }

    Css Position

    No es una técnica de maquetación como tal, pero ayuda a ubicar los elementos en una posición conveniente para nosotros, a través de un sistema de coordenadas. La propiedad que debe modificarse en el elemento es Position

    Sintaxis Visibility:

    Position: static | relative | absolute | fixed | sticky;
    static
    Es el valor predeterminado para todos los elementos de CSS, cuando los elementos tienen fijado este valor, no se les considera posicionados, por lo que algunas animaciones no funcionarían con estos elelementos, El elemento es posicionado de acuerdo al flujo normal del documento. Las propiedades top, right, bottom, left, and z-index no tienen efecto. Este es el valor por defecto.
    Relative
    Con este valor el elemento conserva su lugar en el flujo normal del documento, pero se muestra desplazado, tomando como referencia el espacio donde originalmente debería aparecer con base en los valores de top, right, bottom, and left., este valor no altera la posición de los otros elementos en el documento Este valor crea un nuevo contexto de apilamiento, donde el valor de z-index no es auto.
    Absolute
    Este valor permite al elemento posicionarse dentro del documento, tomando como referencia a su ancestro más cercano que posea un posicionamiento definido, es decir, que no sea static, si no encuentra a este ancestro se posicionará con referencia al <body>, cabe mencionar que el lugar que ocupaba este elemento quedará vacío por lo que será ocupado por el siguiente elemento del documento
    Fixed
    Este elemento se posiciona semejante al valor absolute, la diferencia que la referencia de su posicionamiento será el viewport encima de los demás, lo que permite crear elementos flotantes que permanezcan en la misma posición
    Sticky
    Este valor tiene propiedades tanto de relative como de fixed, ya que mantiene su posición de manera relativa hasta que cruza un umbral que nosotros especificamos al momento de hacer scroll, a partir de ahi se comportará como un elemento fixed al cual se "pegará" hasta que el scroll lo devuelva a su posición inicial, su uso común es crear encabezados o piés de página

    La referencia de la posición de los diferentes elementos se establece a partir de tomar como referencia cualquiera de los 4 costados de los elementos padres, estas propiedades son: top, right, bottom y left.

    El posicionamiento de los elementos puede causar confusión dependiendo de la propiedad donde referenciamos su posición, el sentido positivo es diferente para bottom y top así como para left y right, para ello se usa esta imagen para conocer cual es el sentido positivo de cada una de las referencias.

    Si a un mismo elemento definimos un top y bottom, se toma como referencia a TOP, en el caso que left y right estén especificados gana la dirección de lectura, para nosotros sería LEFT para otros idiomas que tienen el orden de lectura diferente sería RIGHT

    Los elementos se posicionan de acuerdo al flujo normal, al momento de posicionar los elementos, en ocasiones es necesario cambiar el orden como aparecen los objetos, esto se establece a través de la propiedad z-index, entre más grande es el número estará mas al "Frente", entre más pequeño, mas "atrás".

    Hay que tomar en cuenta que el posicionamiento a través del z-index puede verse modificado por el Contexto de apilamiento que es la conceptualización tridimensional de los elementos HTML a lo largo de un eje-Z imaginario relativo al usuario que se asume está de cara al viewport o página web, hay diferentes situaciones que se crean diferentes contextos de apilamiento lo cual afecta la visibilidad de los elementos.
