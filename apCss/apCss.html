Rudimentos

¿Que es CSS?

CSS es el acrónimo de Hojas de estilo en cascada (del inglés Cascading Style Sheet ) es el lenguaje de estilos utilizado para describir la presentación de documentos HTML o XML (en-US) (incluyendo varios languages basados en XML como SVG, MathML o XHTML). CSS describe como debe ser renderizado el elemento estructurado en la pantalla, en papel, en el habla o en otros medios.

CSS es uno de los lenguajes base de la Open Web y posee una especificación estandarizada por parte del W3C. Anteriormente, el desarrollo de varias partes de las especificaciones de CSS era realizado de manera sincrónica, lo que permiía el versionado de las recomendaciones. Probablemente habrás escuchado acerca de CSS1, CSS2.1, CSS3. Sin embargo, CSS4 nunca se ha lanzado como una versión oficial.

En resumen, con CSS podemos definir la manera en que lucirá una página web, desde los colores de los elementos, la tipografía e incluso el orden y el espacio de cada uno de los elementos involucrados.

ALERTA: Para un adecuado entendimiento del manejo de CSS es necesario tener sólidas bases de HTML, de otra forma el aprendizaje será lento, ya que ambas tecnologías se complementan.

CSS en HTML

Una vez que hemos entendido que cada elemento que vamos a modificar debemos definir una regla, ahora debemos entender donde es el lugar adecuado para colocar esa regla y que funcione.

El código CSS se interpreta en el navegador del cliente que abre una página web, para que éste navegador interprete de forma adecuada los estilos que corresponden a un sitio web, debemos especificar como es que cada uno de los elementos que se encuentran dentro de la página deben presentarse y para ello existen 3 formas diferentes de hacerlo.

  • Forma in-line
  • Forma Embebida
  • Forma Enlace
IN-LINE:
Esta forma nos permite colocar el estilo, dentro de los atributos de las etiquetas que se definen en HTML, este atributo solo afecta a la etiqueta en cuestión una a la vez.

Ejemplo:


                                
                                    <p style="font-size: 20px">Mi texto </p>

ALERTA: Trabajar con estilos In-line es cómodo por que se agregan las reglas dentro de los atributos de la etiqueta, sin embargo, no se aprovecha la cascada de CSS, lo cual es una potente herramienta de diseño, tampoco se pueden repetir los estilos a diferentes páginas del mismo proyecto asi mismo afecta la Especificidad, pues el estilo In-line siempre tiene mayor prioridad sobre las otras formas.

EMBEBIDA:
Esta forma nos permite modificar a los elementos semejantes dentro del mismo archivo HTML, pues al usar un selector, este no se encuentra atado dentro de la misma etiqueta, por lo que se puede aprovechar del efecto en cascada y afectar a otros objetos "Semejantes".

Ejemplo:


                                
                                    <head>
                                        <style>
                                            p{
                                                font-size:20px;
                                            }
                                        </style>
                                    </head>

ALERTA: A pesar que la forma embebida nos permite aprovechar de mejor manera la cascada, es una buena práctica utilizar la tercera forma, además que nos permite agregar esas mismas reglas de estilo a otras páginas.

ENLACE:
Esta forma nos permite crear una archivo *.css por separado del archivo *.html, lo que nos permite crear un solo conjunto de reglas que pueden aplicarse a diversas páginas y mantener posteriormente los cambios fácilmente, esta forma de trabajar se considera la mejor práctica para las hojas de estilo.

Ejemplo Enlace:


                                
                                    <head>
                                        <link rel="stylesheet" href="css/estilo.css" >
                                    </head>

Regla CSS

Debemos entender que una página web es una conjunto de objetos acomodados por etiquetas uno después de otro y cada uno de esos objetos generalmente marcados por estas etiquetas contiene propiedades que pueden alterarse y cambiar el aspecto original de la página. Para que podamos visualizarlos de forma correcta, debemos indicar que propiedades deben modificarse.

La forma en que CSS realiza estos cambios es estableciendo "Reglas Predeterminadas" o simplemente "Reglas" las cuales le indican al nevegador como debe mostrar ese elemento que estamos manipulando, cada regla que establezcamos debe tener la siguiente estructura:

A continuación describimos cada una de sus partes:

SELECTOR:
es la manera en la que nos referimos al elemento al cual aplicaremos la regla que estamos definiendo, para ello, todas las declaraciones que afecten a este elemento debemos escribirlas entre llaves "{...}" para mantenerlas agrupadas, existen muchas maneras de referirnos a esos elementos, si los combinamos adecuadamente, podemos acceder a cualquiera de los elementos establecidos en la página.
DECLARACIÓN:
cada una de las declaraciones modifica una propiedad diferente del elemento seleccionado, cada una de ellas se encuentra compuesta por una propiedad y por el valor que esta asumirá, es importante que al finalizar la declaración, ésta termine con ";".
PROPIEDAD:
La propiedad nos indica que característica del elemento queremos cambiar, su anchura, su altura, su color, etc. de esta manera indicamos al navegador como debe visualizarse. es importante que después de mencionar la propiedad separemos con ":" los valores que le daremos a la misma.

NOTA: Cada propiedad puede llevar consigo más de un valor, generalmente, se identifica un valor de otro por que van separados por espacios, de igual manera existen funciones que nos apoyan para colocar de mejor manera estos valores.

VALOR:
Una vez que hemos señalado la propiedad debemos indicar adecuadamente el nuevo valor que ésta tendrá, este cambio de magnitud depende de la propiedad que estamos modificando, una vez que hayamos establecido los valores adecuados, la declaración debe finalizar con ";".

NOTA: Cada Valor es diferente para cada tipo de propiedad, lo más común es que este valor vaya acompañado de un magnitud o unidad de medida que especifica la manera en que la propiedad cambiará.

Herencia, Cascada y Especificidad

Estos 3 conceptos se encuentran estrechamente relacionados y son los responsables que se apliquen o no las reglas que se definen, estas características tambien son útiles para asignarle la misma regla a diferentes elementos con características similares sin necesidad de repetir código

Herencia

La herencia es el mecanismo mediante el cual un objeto padre (o de jerarquía superior) "hereda" características a sus hijos directos y ellos a su vez heredarían éstas características o lo siguientes elementos descendientes. Existirán características que serán heredables y algunas otras no podrán heredarse.

CSS proporciona 4 valores (1 elemento experimental) de propiedad universales especiales para el control de la herencia. Todas las propiedades CSS aceptan estos valores.

inherit:
Este valor hace que la propiedad a la que se aplica, sea exactamente igual al valor de su padre en la misma propiedad, en otras palabras "activa la herencia"
initial:
Cada navegador tiene por defecto una hoja de estilo para los elementos, la cual suele ser diferente para uno de ellos, esta hoja posiciona esos valores básicos, al usar el valor "initial" la propiedad regresa a los valores por defecto que posee el navegador
unset:
Reestablece el valor a su valor natural, puede ser inherit o initial, dependiendo de la propiedad a la que se aplique.
revert:
es un valor experimental, el cual tendría como propósito revertir la cascada y regresar su valor al que heredaría de su ancestro más cercano
all
Esta es una Propiedad abreviada que tiene por objetivo controlar la herencia en las reglas CSS, esta afecta a casi todas las propiedades a la vez y puede tomar cualquiera de los 4 valores anteriores por ejemplo: a {all:revert;}

Cascada

Esta propiedad permite que un regla pueda aplicarse a diferentes elementos, por ejemplo, si definimos una regla para los parrafos "p" todas las etiquetas "p" aplicaran la misma regla, sin embargo, si mas adelante se enuncia una regla diferente para el selector "p", se aplicará la ultima regla que aparezca:

Ejemplo:


                        /*...*/
                        p{color:blue;}
                        p{color:red;}
                        p{color:orange;}
                        /*...*/

En este ejemplo podemos ver visualmente como es que se han aplicado diferentes reglas a las etiquetas "p", sin embargo la regla que prevalecerá será la que tiene color naranja, ya que la cascada respeta la última regla válida.

Especificidad

La especificidad, se aplica para aquellos elementos se encuentran referidos en más de una regla y tienen diferente selector, para ello, se analiza la manera en que el selector se ha construido y se refiere a ese elemento, dándole valores a las diferentes formas de construir el selector, provocando que aquel que sea más específico para referirse al elemento sea quien prepondere sobre los demás.

Para explicar mejor esta idea, se le da un valor numérico a los selectores, se distingue entre etiquetas, clases, id, inline y sobre el important ese valor se suma y se aplica la regla que tenga un selector con mayor número, es decir, mayor especificidad.

Los valores aproximados que se asignan son los siguientes.

Selector Valor Descripcion
*, +, >, ~, " ", ",", :not 0 El selector universal, combinadores y la pseudoclase :not no influyen en el calculo de la especificidad
Tag 1 El selector de tipo de etiqueta "<>", o pseudoelemento "::" agrega una unidad.
Clase 10 El selector de clase ".", de atributo"[]" o pseudoclase ":"
Id 100 Para cada selector de Id "#"
In-Line 1000 Estos estilos se especifican directamente sobre el atributo "style" de la estiqueta correspondiente en el documento html
!important 1,000,000 la declaración que contenga como valor !important prevalece sobre cualquier otra regla

ALERTA: Usar "!important" es una mala práctica, conocer este valor para los atributos es útil para casos extremos, pero no para casos comunes que pueden resolverse con clases o combinadores.

Para entender mejor esta idea, es útil que entendamos como es que el navegador asigna estilos a los diferentes elementos.

  1. Herencia
  2. Orden del código
  3. Especificidad
  4. Importancia
Herencia:
El navegador primero se fija en la hoja de estilos que tiene predeterminada y la forma en que estos estilos se heredan a sus hijos, estos se colocan de forma inicial
Órden del código:
Después observa los estilos que se encuentran definidas en la hoja de estilos del usuario, si hay conflicto, las hojas definidas por el usuario tienen preponderancia, si hay diferentes reglas que se aplican al mismo elemento se observa el orden en que aparecen y las últimas definidas son las que tienen mayor preponderancia
Especificidad:
Si existen coincidencias de reglas que se refieren a un mismo elemento, el navegador observa que regla se ha definido de forma "más específica" es decir, cual regla provoca que se seleccione a un elemento de manera inequivoca o de forma menos general.
Importancia
Propiedades que contengan el valor !important

SELECTORES

Un selector es la manera con la cual nosotros nos referimos a un elemento de la página web a la cual nosotros aplicaremos una serie de declaraciones que modificarán su aspecto final, es importante conocer la forma de dirigirnos a ese elemento, por que si lo hacemos de forma incorrecta, los estilos no se aplicaran correctamente, para ello, existe un amplia diversidad de herramientas que podemos utilizar y combinar entre ellas para seleccionar correctamente el elemento.

A continuación enumeramos los tipos de selectores que disponemos.

  • Selector de Tipo
  • Selector de Clase
  • Selector de ID
  • Selector de Atributo
  • Selector de Pseudoclases
  • Selector de Pseudoelementos
  • Selector Universal

De la misma forma disponemos de operadores de combinación, los cuales nos permiten ampliar de forma significativa nuestras herramientas de selección, entre ellas encontramos:

  • Combinador coma ","
  • Combinador Espacio" "
  • Combinador suma"+"
  • Combinador virgulilla"~"

Entendamos como funcionan cada uno de ellos:

SELECTOR DE TIPO
Un selector de tipo también recibe el nombre de selector de nombre, de etiqueta, selector de elemento o selector general porque selecciona un elemento/etiqueta HTML del documento de forma directa.

En archvo HTML:


                                
                                    <p>Mi texto </p>

En archvo CSS:


                                
                                    p{
                                      font-size: 20px;
                                    }

ALERTA: cuando usamos esta forma de seleccionar los elementos, la regla que definamos se aplica para todas las etiquetas del mismo tipo. En el caso del ejemplo todas las etiquetas "p" seran afectadas por este estilo.

SELECTOR DE CLASE
Este selector de clase nos proporciona un metodo para seleccionar a solo "algunos" elementos a quienes deseamos aplicar la regla, es decir, solo los elementos que pertenecen a la clase que definimos pueden ser seleccionados, en este caso, no importa si los elementos son de etiquetas diferentes o etiquetas iguales.

La forma correcta de implementarlo es asignadole una clase en el atributo "class" en las etiquetas del archivo *.html de esa manera "marcamos" las etiquetas a las cuales se le aplicaran la regla que definiremos en el archivo *.css, cuando definamos la regla el nombre de la clase debe estar precedida de "." para indicar que es el nombre de la clase.

En archvo HTML:


                                
                                    <p class="amarillo"> Mi texto </p>

En archvo CSS:


                                
                                    .amarillo{
                                      color: yellow;
                                    }
De ser necesario, podemos modificar el ejemplo para asignar la clase a diferentes etiquetas y observar como se aplica la regla a las etiquetas que seleccionamos, miestras que las etiquetas que no contienen a la clase, quedan excluidas de la regla, todo esto se aplica sin modificar el archivo *.css.

En archvo HTML:


                                
                                    <h1 class="amarillo"> Titulo Amarillo </h1>
                                    <p> Este parrafo va en color predeterminado</p>
                                    <p> Este parrafo tampoco se aplica la regla/p>
                                    <p class="amarillo"> Este parrafo SI es amarillo</p>
De la misma forma podemos agregar más de una clase a las etiquetas, lo que nos permite agregar más características a un elemento. eso lo podemos ver en este ejemplo:

En archvo HTML:


                                
                                    <!-- este titulo llevaria caja alrededor y letra amarilla-->
                                    <h1 class="amarillo caja"> Titulo Amarillo </h1> 
                                    <p> Este parrafo va en color predeterminado</p>
                                    <p class="caja"> Este parrafo tampoco se aplica la regla pero 
                                    tiene un borde negro alrededor</p>
                                    <p class="amarillo"> Este parrafo SI es amarillo</p>

En archvo CSS:


                                
                                    .amarillo{
                                      color: yellow;
                                    }
                                    .caja{
                                        border: thin solid black;
                                    }
SELECTOR DE ID
El Selector Id o de referencia absoluta nos ayuda a seleccionar directamente un único elemento que en su etiqueta html contenga el atributo "id". para referirnos desde el archivo *.css es necesario que se anteponga el caracter "#"

En archvo HTML:


                                
                                    <p id="amarillo"> Mi texto </p>

En archvo CSS:


                                
                                    #amarillo{
                                      color: yellow;
                                    }
SELECTOR DE ATRIBUTO
El Selector de Atributo nos permite elegir un elemento de la página web al cual le aplicaremos una regla, siempre y cuando se haya especificado por lo menos algún atributo, podemos seleccionar el elemento por el tipo de atributo, por el valor preciso del atributo o por alguna frase que pertenezca al valor del atributo, la nomenclatura es la siguiente:

Presencia y Valor
Selector Ejemplo
[attr] 
a[title]{/*...*/}
DESCRIPCION: Selecciona a los elementos o etiquetas del archivo *.html que contengan un atributo concreto, sin importar su valor. reemplazamos attr por el nombre del atributo que buscamos entre los corchetes.
[attr=value] 
                                            
                                                a[href="https://example.com"]
                                                {/*...*/}
DESCRIPCION: Selecciona elementos que tiene un atributo de nombre attr, y su valor exacto es value (la cadena de caracteres que se indica entre corchetes).
[attr~=value] 
p[class~="special"]{/*...*/}
DESCRIPCION: Se lecciona a las etiquetas del archivo *.html con el atributo de nombre attr, cuyo valor es exactamente value, o los elementos con un mismo atributo attr que contiene uno o más valores de los cuales, al menos uno, coincide con value. Hay que tomar en cuenta que los diferentes valores generalmente se separan por espacios.
[attr|=value] 
div[lang|="zh"]{/*...*/}
DESCRIPCIÓN: Relaciona a los elementos con un nombre de atributo, attr, cuyo valor puede ser exactamente value o puede comenzar con value seguido inmediatamente por un guion.


Coincidentes con subcadenas
Selector Ejemplo
[attr^=value] 
li[class^="box-"]{/*...*/}
DESCRIPCIÓN: Relaciona a los elementos con un nombre de atributo, attr, cuyo valor comienza exactamente con value.
[attr*=value] 
li[class*="box"]{/*...*/}
DESCRIPCIÓN: Relaciona a los elementos con un nombre de atributo, attr, cuyo valor contiene al menos una ocurrencia de la subcadena value.
[attr$=value] 
li[class$="-box"]{/*...*/}
DESCRIPCIÓN: Relaciona a los elementos con un nombre de atributo, attr, cuyo valor termina exactamente con la subcadenavalue.


Case sensitive
Selector Ejemplo
i 
li[class^="a" i]{/*...*/}
DESCRIPCIÓN: Selecciona los elementos que cumplan con la descripción del selector pero adicionalmente su valor sea en letras minusculas
s 
ol[type="A" s]{/*...*/}
DESCRIPCIÓN: Selecciona los elementos que cumplan con la descripción del selector pero adicionalmente su valor sea en letras minusculas, este atributo es experimental.
SELECTOR DE PSEUDOCLASE
Las Pseudoclases son palabras claves que se añaden a los selectores, los cuales representan un estado en particular del elemento seleccionado. Todas las pseudoclases se comportan del mismo modo. Seleccionan un fragmento del documento que está en un estado determinado y se comportan como si se hubiera añadido una clase a su html.

Ejemplo

                                
                                    article p:first-child{/*...*/}
                                    a:hover{/*...*/}
                                    a:visited{/*...*/}
Algunas pseudoclases utiles son las siguientes:
  • :nth-child(): Permite seleccionar un hijo de un elemento que tiene varios hijos, cada hijo se enumera según el orden de aparición, lo que nos permite colocar ese número ordinal entre paréntesis para ubicarlo, también es posible usar las palabras "odd" y "even" para indicar elementos impares o pares, según sea necesario
  • :first-child: Selecciona al primer hijo de un elemento
  • :last-child: Selecciona al último hijo
  • :only-child: Selenciona al que tiene solo un hijo
  • :root: Representa a la etiqueta Html, pero tiene mayor especificidad, por lo cual predomina. generalmente se usa para declarar variables.
  • :hover: se activa al pasar el mouse por encima
  • :active: se usa este estado en el momento que se mantiene presionado el mouse, para que funcione debe ir en orden las reglas con las pseudoclases, :link{}, :visited{}, :hover{}, :active
  • :target
  • :not
  • :empty
SELECTOR PSEUDOELEMENTO
Los Pseudoelementos se comportan como si agregaramos un elemento html nuevo al marcado, para diferenciarlos de las pseudoclases agregamos "::" al selector y el nombre del pseudoelemento en estaLista de Pseudoelementos y Pseudoclases podemos ver una lista de ellos.

Ejemplo

                                
                                    article::before{/*...*/}
                                    p::after{/*...*/}
                                    p::first-letter {/*...*/}
                                    a::first-line{/*...*/}

ADVERTENCIA: para que podamos usar un ::before o un ::after debemos agregar un valor a "content" aunque sea un valor como content:""; para que pueda verse el elemento y las modificaciones que le hagamos a estos pseudoelementos. Estos pseudoelementos no funcionan en los elementos in-line

SELECTOR UNIVERSAL
El selector universal se indica con un asterisco (*) y selecciona todos los elementos del documento (o del elemento padre si está encadenado con otro elemento y un operador de combinación descendente, por ejemplo). *{margin: 0;} Es posible observar este tipo de comportamiento en las «hojas de estilo de puesta a cero» (o «hojas de estilo reset»), que anulan el formato del navegador.

Uno de los usos del selector universal es facilitar la legibilidad de los selectores y clarificar sus funciones. Por ejemplo, si quiero seleccionar el primer elemento hijo de cualquier elemento <article> y poner ese elemento, cualquiera que sea, en negrita, puedo utilizar el selector :first-child, que veremos con mayor detalle más adelante en el artículo de pseudoclases y pseudoelementos, como selector descendente junto con el selector de elemento <article>:

article :first-child {/*...*/}

Sin embargo, esto podría confundirse con article:first-child, que selecciona cualquier elemento <article> que sea el primer elemento hijo de otro elemento.

Para evitar esta confusión podemos añadir al selector :first-child el selector universal. De este modo la función del selector resulta obvia: seleccionará cualquier elemento que entre en la jerarquía de primer hijo de un elemento <article>:

article *:first-child {/*...*/}
OPERADORES DE COMBINACION
Los operadores de Combinación nos permiten combinar diferentes selectores e identificar la relacion que hay entre ellos

Operadores de Combinación
Operador Ejemplo
"," 
div, span{/*...*/}
DESCRIPCIÓN: En este caso la "," nos permite agrupar diferentes bloques que comparten la misma regla, todas las declaraciones se aplican al "div" y a "span" sin importar la relacion que tengan entre si
" " 
div span{/*...*/}
DESCRIPCIÓN:El espacio en blanco Selecciona todos los elementos "span" que se encuentran dentro de un "div", estos elementos no importa si son hijos directos o "nietos", siempre y cuando "div" sea el ancestro que los contenga
">" 
div>span{/*...*/}
DESCRIPCIÓN:El selector hijo directo ">" permite seleccionar a los elementos "span" que son hijos directos o se encuentran exactamente en el siguiente nivel de jerarquia de "div", en esta ocasión los elementos descendientes que se encuentren en niveles inferiores, no se veran afectados
"+" 
div+span{/*...*/}
DESCRIPCIÓN:El selector Hermano adyacente "+" selecciona al "span" que es hermano de "div" y que son contiguos, es decir, esxactamente a lado de "div", en este caso ambos comparten la misma jerarquía
"~" 
div~span{/*...*/}
DESCRIPCIÓN:El selector Hermano siguiente "~" selecciona al o los "span" que son hermanos de "div" no importa si son contiguos o hay varios bloques de diferencia siempre y cuando se encuentren en el mismo nivel de jerarquia y se encuentren despues de div
